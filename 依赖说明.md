# 极客 OCR v4.0 运行时依赖说明

本程序采用 **框架依赖部署**（`SelfContained=false` + `WindowsAppSDKSelfContained=false`），不附带运行时，体积小（~180KB exe + 依赖 dll）。运行需目标机已安装下列运行时。

---

## 一、终端用户运行时依赖

| 依赖 | 版本 | 下载地址 |
|------|------|---------|
| **.NET Desktop Runtime** | 10.0.x | https://dotnet.microsoft.com/download/dotnet/10.0 |
| **Windows App SDK Runtime** | 1.7 | https://aka.ms/windowsappsdk/1.7/latest |

### 自动检测机制

- **Windows App SDK 1.7 缺失**：程序启动时 `Bootstrap.TryInitialize` 检测，**弹框提示并自动打开下载页**（浏览器跳转）。
- **.NET 10 Desktop Runtime 缺失**：exe（apphost）无法启动，Windows 系统**自动弹框**提示安装 .NET（含下载链接）。

### 安装顺序（缺一不可）

1. 先装 **.NET 10.0 Desktop Runtime**
2. 再装 **Windows App SDK 1.7 Runtime**
3. 重新双击 `OcrTranslator.exe`

### 系统要求

- Windows 10 1809 (17763) 及以上 / Windows 11
- x64 架构（另提供 x86、ARM64 编译配置）

## 二、开发者编译依赖

| 工具 | 用途 | 说明 |
|------|------|------|
| **Visual Studio 2022+**（含「.NET 桌面开发」工作负载）| 提供 MSBuild 编译 WinUI 3 | ⚠️ **不要用 `dotnet build`**，其 XamlCompiler 遇 XAML 错误会静默崩溃，必须用 VS MSBuild |
| **.NET 10 SDK** | 编译 | 10.0.301+ |

编译方式：
- 双击 `build.cmd`（自动用 vswhere 定位 VS MSBuild）
- 或 VS 打开 `OcrTranslator.csproj`，F5 运行
- 或命令行：
  ```
  "C:\Program Files\Microsoft Visual Studio\18\Community\MSBuild\Current\Bin\MSBuild.exe" OcrTranslator.csproj -p:Configuration=Release -p:Platform=x64
  ```

打包发布：双击 `publish.ps1` → 产出 `dist\OcrTranslator_v4_x64.zip`

## 三、为什么是框架依赖（而不是自包含）？

| 对比 | 框架依赖（当前）| 自包含 |
|------|----------------|--------|
| 包体积 | ~180KB exe + dll | ~174 MB（含完整运行时）|
| 目标机依赖 | 需装 .NET 10 + WinAppSDK 1.7 | 零依赖 |
| 跨 Windows 版本 | 依赖本机运行时，版本永远匹配，不会因系统 DLL 版本冲突崩溃（0xc0000602）| 自带系统 DLL，跨版本易冲突 |

v3 文档（`打包问题根因分析与修复方案.md`）验证：自包含模式（`WindowsAppSDKSelfContained=true`）会带入 7 个系统级 DLL，跨 Windows 版本触发 `0xc0000602` 崩溃；框架依赖从系统加载，**版本永远匹配**，最稳。
